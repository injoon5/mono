Mono is pure CSS — no JavaScript dependencies, no build steps. It uses modern CSS features like custom properties, oklch color space, and logical properties. The only external asset is the Pretendard font served from jsDelivr.
Yes. Dark mode is supported via both prefers-color-scheme media query for system-level preference, and a data-theme="dark" attribute on the root element for manual toggling. The color palette inverts cleanly across all components.
Override or extend the custom properties in :root to change the design tokens globally. Add new component classes by following the existing naming patterns. The framework is intentionally low-specificity so project styles can override without !important.
Focus states are visible and use a consistent outline style. Form controls use native HTML elements where possible. All interactive elements are keyboard-navigable. ARIA attributes should be added by the consumer where needed for dynamic components.
